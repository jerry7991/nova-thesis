# Code of Conduct

## Our Pledge

nova-thesis exists to make software better by challenging implementations before they ship. Everyone who contributes — scenarios, postmortems, skill improvements, or code — is welcome here regardless of experience level, background, or the number of production incidents they've caused.

## Our Standards

**Constructive behavior:**
- Submitting scenarios grounded in real failure modes
- Citing real postmortems with accurate links
- Giving honest, specific feedback on PRs
- Accepting that your scenario might need revision

**Not acceptable:**
- Submitting fabricated postmortems or fake incident links
- Dismissing someone's contribution without explanation
- Using the framework to shame or blame individuals for past incidents

## Scope

This Code of Conduct applies to the GitHub repository, issues, PRs, and any discussions related to nova-thesis.

## Enforcement

Report concerns to [@jerry7991](https://github.com/jerry7991) via GitHub. All reports will be reviewed and responded to.
